## Česká verze

### Jak přispět

Vítáme všechny druhy příspěvků:

- 🐛 **Hlášení chyb** - pokud najdete problém, otevřete issue
- ✨ **Nové funkce** - návrhy na vylepšení jsou vítány
- 📝 **Dokumentace** - opravy překlepů, vylepšení textů
- 🔧 **Kód** - opravy chyb, nové funkce, optimalizace
- 🌍 **Překlady** - pomoc s jinými jazyky

### Proces přispívání

1. **Fork** repozitáře
2. **Vytvořte branch** pro vaši funkci (`git checkout -b feature/AmazingFeature`)
3. **Proveďte změny** s jasnými commit zprávami
4. **Otestujte** své změny
5. **Push** do vašeho forku (`git push origin feature/AmazingFeature`)
6. **Otevřete Pull Request**

### Pravidla pro Pull Requests

- ✅ Jasný popis změn
- ✅ Otestované na skutečném hardware (pokud je to možné)
- ✅ Dodržení stávajícího stylu kódu
- ✅ Aktualizace dokumentace (pokud je třeba)
- ✅ Jeden feature per PR

### Coding Standards

- Používejte **čitelný a komentovaný kód**
- Dodržujte **ESPHome best practices**
- Testujte změny na **skutečném ESP32**
- Dokumentujte **nové funkce**

### Hlášení chyb

Při hlášení chyby prosím uveďte:

- **Popis problému** - co se děje
- **Očekávané chování** - co by se mělo stát
- **Kroky k reprodukci** - jak problém vyvolat
- **Verze firmware** - jakou používáte
- **Hardware** - ESP32, HX711, load cells
- **Logy** - pokud jsou k dispozici

### Návrhy funkcí

Pro návrhy nových funkcí:

- Popište **use case** - proč je funkce užitečná
- Navrhněte **implementaci** - jak by to mohlo fungovat
- Zvažte **kompatibilitu** - nebude to nic rozbíjet

## English Version

### How to Contribute

We welcome all types of contributions:

- 🐛 **Bug reports** - if you find an issue, open an issue
- ✨ **Feature requests** - improvement suggestions are welcome
- 📝 **Documentation** - typo fixes, text improvements
- 🔧 **Code** - bug fixes, new features, optimizations
- 🌍 **Translations** - help with other languages

### Contribution Process

1. **Fork** the repository
2. **Create a branch** for your feature (`git checkout -b feature/AmazingFeature`)
3. **Make changes** with clear commit messages
4. **Test** your changes
5. **Push** to your fork (`git push origin feature/AmazingFeature`)
6. **Open a Pull Request**

### Pull Request Guidelines

- ✅ Clear description of changes
- ✅ Tested on real hardware (if possible)
- ✅ Follow existing code style
- ✅ Update documentation (if needed)
- ✅ One feature per PR

### Coding Standards

- Use **readable and commented code**
- Follow **ESPHome best practices**
- Test changes on **real ESP32**
- Document **new features**

### Bug Reports

When reporting a bug, please include:

- **Problem description** - what happens
- **Expected behavior** - what should happen
- **Steps to reproduce** - how to trigger the issue
- **Firmware version** - which one you're using
- **Hardware** - ESP32, HX711, load cells
- **Logs** - if available

### Feature Requests

For new feature suggestions:

- Describe the **use case** - why the feature is useful
- Propose an **implementation** - how it could work
- Consider **compatibility** - won't it break anything
