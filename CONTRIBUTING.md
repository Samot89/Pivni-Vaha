# 🤝 Contributing

Thank you for your interest in contributing to the Beer Keg Scale project!

[🇬🇧 English](#english) | [🇨🇿 Česky](#česky)

---

## English

### Ways to Contribute

- 🐛 **Bug reports** – open an issue if you find a problem
- ✨ **Feature requests** – suggestions for improvements are welcome
- 📝 **Documentation** – fix typos, improve explanations
- 🔧 **Code** – bug fixes, new features, optimisations
- 🌍 **Translations** – help with other languages

### Contribution Process

1. **Fork** the repository
2. **Create a branch** for your feature (`git checkout -b feature/AmazingFeature`)
3. **Make changes** with clear commit messages
4. **Test** your changes (on real hardware if possible)
5. **Push** to your fork (`git push origin feature/AmazingFeature`)
6. **Open a Pull Request**

### Pull Request Guidelines

- ✅ Clear description of what changed and why
- ✅ Tested on real hardware (if applicable)
- ✅ Follows existing code style
- ✅ Documentation updated (if needed)
- ✅ One feature / fix per PR

### Coding Standards

- Write **readable, commented code**
- Follow **ESPHome best practices**
- Test changes on a **real ESP32**
- Document **new features**

### Bug Reports

Please include:
- **Problem description** – what happens
- **Expected behaviour** – what should happen
- **Steps to reproduce** – how to trigger the issue
- **Firmware version**
- **Hardware** – ESP32, HX711, load cells
- **Logs** – if available

### Feature Requests

Please include:
- **Use case** – why the feature is useful
- **Proposed implementation** – how it could work
- **Compatibility** – does it break anything?

## Česky

### Způsoby přispívání

- 🐛 **Hlášení chyb** – otevřete issue
- ✨ **Návrhy funkcí** – vylepšení jsou vítána
- 📝 **Dokumentace** – opravy překlepů, vylepšení textů
- 🔧 **Kód** – opravy chyb, nové funkce
- 🌍 **Překlady** – pomoc s dalšími jazyky

### Postup přispívání

1. **Forkněte** repozitář
2. **Vytvořte branch** (`git checkout -b feature/MojeFunkce`)
3. **Proveďte změny** s jasnými commit zprávami
4. **Otestujte** změny (pokud možno na reálném HW)
5. **Pushněte** do forku (`git push origin feature/MojeFunkce`)
6. **Otevřete Pull Request**
